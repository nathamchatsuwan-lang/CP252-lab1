# Requirement Document
## Functional Requirement
1. ต้องการ user account เพื่อจอง
2. ต้องการแสดงจำนวนที่จอดรถคงเหลือแบบเรียลไทม์
3. ต้องการฟีเจอร์ Check in, Check-out เพื่อแสดงที่จอดรถแบบเรียลไทม์ได้ตรง
4. ต้องการให้มีระบบแบนเป็นวันหรือจ่ายค่าธรรมเนียมเพิ่มเติมสำหรับผู้จอดล่วงเวลา
5. ต้องการ การรองรับการชำระเงินออนไลน์และบัตรเครดิต/เดบิต
6. ต้องการส่งแจ้งเตือนให้ผู้ใช้เพื่อยืนยันการจอง, เตือนเวลาใกล้หมดจอง, การยกเลิก etc.
7. ต้องรองรับการยกเลิกการจองและเปลี่ายนสถานะที่จอดให้ว่างโดยอัตโนมัติ พร้อมการคืนเงินตามกฎที่กำหนดไว้
8. ต้องการดาต้าเบสในการจัดเก็บข้อมูลโดยรวมของผู้ใช้
9. ต้องการ การคำนวณจำนวนที่ต้องชำระอัตโนมัติโดยอิงจากประเภทลานจอดรถและเวลาเป็นต้น
10. ต้องการให้แอพรองรับที่จอดรถพิเศษเช่น Super Car, Big Bike etc.
## Non-Functional Requirement
1. หน้าแสดงที่จอดรถและการค้นหาต้องตอบสนองไว
2. ต้องพร้อมใช้งานตลอด ไม่ค่อยมีเวลา Maintance ที่นานเกินไป
3. รองรับลูกค้าที่มากขึ้น ไม่ให้การดึงข้อมูลจากเซิร์ฟเวอร์กระตุก
4. ทำให้ข้อมูลเรียลไทม์และตรงมากที่สุด
5. ดูแล/รักษาความปลอดภัยข้อมูลส่วนบุคคล
6. ใช้งานง่าย
7. เข้ากันได้กับหลายแพพลตฟอร์ม
8. มีการสำรองฐานข้อมูล
9. มีความยืดหยุ่นในการตั้งค่าเช่น ui
10. ตามรอยข้อผิดพลาดได้ง่าย เพื่อการแก้ไขที่สะดวกสบาย
